# RAG 检索增强生成

RAG（Retrieval-Augmented Generation，检索增强生成）是一种把**外部知识检索**与**大语言模型生成**结合的范式。模型回答前，先从知识库召回相关片段，再把片段作为上下文交给 LLM 生成答案。

> [!note] 为什么需要 RAG
> 大模型的参数知识是静态的、容易过时，也无法覆盖私有内容。RAG 让模型"开卷答题"，既能引用最新内容，又能给出来源，降低幻觉。

## 核心链路

1. **切分（Chunking）**：把 Markdown 笔记拆成语义片段。
2. **嵌入（Embedding）**：用 [[embedding-模型|嵌入模型]] 把片段转成向量。
3. **召回（Retrieve）**：按问题向量做相似度检索，取 top-k 片段。
4. **生成（Generate）**：把片段拼进 prompt，交 LLM 流式输出。

本项目的检索与索引基于 LlamaIndex，向量存储用 Chroma，详见 [[blog-架构]]。

> [!tip] 引用来源
> 每次回答都会附上召回片段对应的原文链接，方便读者回溯到 [[quartz-静态博客]] 渲染的页面。
